# PE Release Preparation

> **Related**: [Takeoffs](/workflows/fabrication-engineer/takeoffs.html) | [FE to PE Release](/workflows/fe-to-pe-release.html) (checklist and procedure)

Use this page as a **prep outline**; the authoritative pre-release requirements and step-by-step SOP live on [FE to PE Release](/workflows/fe-to-pe-release.html).

## Material Checks

- Confirm all materials, hardware, and vendor work are ordered and received.
- Before release, BOM lines, **Fixed QTY**, takeoffs, and part masters must match the model—see [FE to PE Release — checklist (BOM/ERP)](/workflows/fe-to-pe-release.html#checklist-bom-erp) and [Epicor Job Management](/tools/epicor/job-management.html).

## Picking Up Redlines

- Apply architect redlines to shop drawings.
- Final set becomes "For Record" sent back to architect.

## Finalizing Shop Drawings

- Ensure completeness: materials, hardware, sizes, build strategy, SCs.
- Follow [Rhino Drafting](/standards/rhino-drafting/) standards.
- Submit for approval using the [Approvals Process](/tools/approvals-process.html).
- Approval prerequisites (PA, client submittal) are listed in [FE to PE Release — checklist](/workflows/fe-to-pe-release.html#approval-prerequisites).

## Creating PE File

- Clean release file: no clutter, invalid blocks, or empty layers—see [Rhino geometry and hygiene](/workflows/fe-to-pe-release.html#1-rhino-model-geometry-and-hygiene) on the FE→PE page.

## Released in Epicor

- Confirm job release, SC upload (**UD40** / **UD37** for CMG), and job-level fields per [FE to PE Release](/workflows/fe-to-pe-release.html) before notifying the team.

## Notify PE Team

- Post release in CO Production Eng chat.
- Include links to all relevant files.
- Use the FE→PE form and PM tag as described in [Communication and handoff meeting](/workflows/fe-to-pe-release.html#communication-handoff-meeting).
