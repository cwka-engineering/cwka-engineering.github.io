## Cartooning a Drawing Set

Cartooning is the practice of planning the full sheet structure of a drawing set before detailed work begins. It produces a sheet-by-sheet outline — page numbers, titles, and a rough indication of which views and details will appear on each page — that serves as the roadmap for the entire documentation effort.

**Why cartoon?**

- Forces early identification of what coverage the scope actually requires
- Exposes gaps (missing sections, unresolved details, unstated hardware) before modeling is underway
- Enables workload distribution across multiple engineers on the same project
- Gives the lead engineer and PM a clear picture of documentation scope and progress

**When to cartoon:**

Cartooning is most valuable on large or multi-engineer scopes where the drawing set has significant depth or where work will be handed between engineers. For smaller, straightforward scopes a mental outline may suffice — but any scope where you would hesitate to describe the full set off the top of your head benefits from a written cartoon.
