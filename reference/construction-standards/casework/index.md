# Casework Construction Standards

Standard construction rules for CWK/DFW casework: carcase assembly, the 32mm shelf-pin system, cabinet-family-specific dimensions, drawers, fillers, and CNC boring patterns. All casework conforms to **Architectural Woodwork Institute (AWI) Custom or Premium grade** — see [AWI Standards Reference](/reference/awi-standards.html) for the quality-grade requirements this construction spec is built to satisfy.

> **Source**: CWKA Standard Casework Construction Manual v2, extracted from the issued drawing set (`CWKA_ENG_Standard_Casework.pdf`, 17 pages, 28 drawings) and reconciled against a lead-engineer field-verification pass.

## Available Pages

- **[Carcase Assembly & 32mm System](/reference/construction-standards/casework/carcase-and-line-bore.html)** — Universal rules that apply to every cabinet type: structural members, back panels, shelves, predrills, and shelf-pin line boring
- **[Base Cabinets](/reference/construction-standards/casework/base-cabinets.html)** — Dimensions, stretcher layout, doors, hinges, pulls, dividers, type-specific notes, and the ladder base platform
- **[Drawer Construction](/reference/construction-standards/casework/drawer-construction.html)** — Drawer front/box/slide/pull specs, graduated drawer heights, and pencil drawers
- **[Upper Cabinets](/reference/construction-standards/casework/upper-cabinets.html)** — Key differences from base cabinets: nailers-only construction, flush doors, bottom-mounted pulls
- **[Tall Cabinets](/reference/construction-standards/casework/tall-cabinets.html)** — Fixed-shelf requirement and dual line-bore zones
- **[Fillers](/reference/construction-standards/casework/fillers.html)** — L-cleat filler construction, corner fillers, ADA filler

## How to read these pages: confidence levels

This spec was built by extracting rules from drawing sheets and reconciling them against a lead engineer's (MH) field-verification markup. Not every rule carries the same level of certainty, and this matters — using an unconfirmed dimension in a cut list or Epicor part record is a real production risk. Every page in this section uses one of these callout types, and none of them should be read as interchangeable:

> **Confirmed:** Directly cross-checked against a specific drawing sheet. Highest confidence.

> **Field-verified:** A correction or addition from MH's markup that wasn't independently visible in the available 2D drawings, but is trusted on the engineer's authority. Not drawing-verifiable, but treated as current and correct.

> **Note:** An exception, boundary condition, or clarification — not a confidence flag, just a rule you need to know.

Where a rule was corrected between v1 and v2 of the source manual, the page says so explicitly rather than silently presenting the new value — if you're cross-referencing an old cut sheet or a memory of the v1 numbers, the callout will tell you what changed and why.

There is no "unconfirmed, do not use" tier remaining in this section — the three items that were genuinely open in the source manual (BCT part-number rename, UC2D divider convention, upper filler depth formula) have since been confirmed as intentional and are documented as **Field-verified** on their respective pages, each with a note that the naming/drawing has not yet been updated to match.
